----	Starten des Programms	----


Zum Starten des Programms je nach Betriebssystem eine der folgenden Dateien ausführen

Windows: run.bat
Linux: run.sh
MacOs: run.sh

Eventuell müssen für die run.sh Dateien noch weitere Konfigurationen gemacht werden
Ansonsten kann das Programm auch in der Kommandozeile mit: " java -jar TDDT.jar " gestartet werden



----	notwendige Vorbereitungen vor Programmstart	----


Vor Programmstart müssen die Übungsdateien in den Ordner "Ubungen" kopiert werden.
Die Form der Übungsdateien kann in den beigefügten Beispieldateien ersehen werden.

!!! Achtung: Es darf keine Übungdatei "Test.txt" genannt werden !!!

Nun eine etwas ausführlichere Erklärung zum Hauptprogramm:

Der Ablauf des Hauptprogramms ist in mehrere "Phasen" unterteilt.

Die einzelnen Phasen sind:
-Red
-Green
-Refactor

In Red wird der Benutzer aufgefordert Tests zu schreiben, wobei mindestens einer dieser Tests fehlschlagen muss.
Wenn diese Bedingung erfüllt ist kann zu Green gewechselt werden.
In Green wird der zu testende Code geschrieben.
Der Benutzer kann nach Wunsch wieder zurück zu Red wechseln.
Wenn alle Tests erfolgreich sind kann der Benutzer zu Refactor wechseln.
In Refactor kann der Code aus Green verbessert werden.
